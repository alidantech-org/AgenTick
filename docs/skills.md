# AgenTick Skill Library Design

AgenTick adopts the open Agent Skills directory format. A source skill is a
folder containing a required `SKILL.md` with YAML frontmatter and optional
`scripts/`, `references/`, and `assets/` resources.

## Project layout

```text
agents/
├── skills.yml             # user intent: skill ids and requested versions
├── skills.lock.yml        # exact registry result and SHA-512 SRI integrity
├── skills/                # project-owned skill source, publishable by the user
└── skillib/               # registry-installed, generated, read-only, Git-ignored
```

No skill file, lock file, or AgenTick configuration is required at repository
root. The CLI discovers the Git root and then checks only its `agents/` folder.

## Declaration versus lock

`skills.yml` is edited intentionally by the project owner:

```yaml
version: 1
skills:
  - id: alidantech/typescript-review
    version: ^1.2.0
    enabled: true
```

`skills.lock.yml` is generated by `agentick pull` and committed:

```yaml
lockfileVersion: 1
registry: https://skills.example.com
skills:
  alidantech/typescript-review:
    version: 1.2.4
    integrity: sha512-...
    resolved: https://skills.example.com/api/v1/skills/alidantech/typescript-review/versions/1.2.4
```

The lock is reproducibility evidence. Exact version, configured registry origin,
resolved endpoint, and SHA-512 integrity must all match before installation.

## Registry transport

Skills are ordinary directories after installation. The network transport is a
deterministic JSON bundle with sorted paths and base64 file contents. AgenTick:

- rejects absolute paths, backslashes, NUL bytes, `..` traversal, duplicates, and
  symlinks;
- enforces file count and byte limits;
- requires `SKILL.md` and validates its metadata;
- checks that the skill name matches the registry id;
- computes SHA-512 over canonical bundle JSON;
- installs through a temporary directory and atomic rename;
- marks installed files read-only;
- never automatically executes downloaded scripts.

## Trust model

Public skills may be pulled anonymously. Private pulls and all pushes require a
revocable scoped account token. A registry URL must come from the environment
variable named in `agents/agentick.yml`; redirects and resolved URLs are not
allowed to leave that configured origin.

A coding AI may read an installed skill. It must not edit the generated library,
change its lock, publish a skill, or execute bundled scripts without the normal
project permissions and human approval.
